# Corivo 触发决策系统设计

## 概述

让 Corivo 自己判断什么时候需要告诉用户什么，而不是被动等待用户查询。

## 核心哲学

> Corivo 是有生命的，它会"思考"什么时候需要提醒用户

- **主动**：不是用户问才答，而是主动提醒
- **克制**：不是每次都推送，避免打扰
- **智能**：基于上下文和状态判断推送时机

## 触发规则

### 1. 冲突提醒 (Conflict)

```
触发条件：检测到新的矛盾关联
优先级：P0 - 立即推送
克制：同一矛盾只提醒一次

示例：
[corivo] ⚡ 与之前的决策矛盾
  之前：决定用 TypeScript
  现在：代码中使用 JavaScript
```

### 2. 遗忘提醒 (Forgotten)

```
触发条件：重要决策 3-7 天未访问
优先级：P1 - 每天检查，最多 1 条
克制：已经提醒过的不重复

示例：
[corivo] 🌱 考虑一下 Rust 迁移
  决策于 3 天前，还未开始实施
```

### 3. 上下文提醒 (Relevant)

```
触发条件：用户正在讨论相关话题，且有相关经验
优先级：P2 - 适时推送
克制：同一会话不重复，每次最多 1 条

示例：
[corivo] 💡 你之前决定不用 React
  当前对话中提到 "前端框架"
```

### 4. 需要关注 (Attention)

```
触发条件：重要记忆进入 cooling/cold 状态
优先级：P3 - 定期检查
克制：每次最多 2 条

示例：
[corivo] ⚠️ 3 条记忆需要关注
  API 密钥决策 (生命力: 35)
  数据库设计 (生命力: 20)
```

### 5. 周总结 (Summary)

```
触发条件：每 7 天
优先级：P4 - 定期推送
克制：只在周一或长时间未登录后

示例：
[corivo] 📊 本周总结
  新增 12 条决策
  完成 3 个任务
  2 个决策需要跟进
```

## 克制机制

### 推送频率控制

| 触发类型 | 最大频率 | 窗口期 |
|---------|---------|--------|
| 冲突 | 立即 | 同一矛盾 1 次 |
| 遗忘 | 1 条/天 | 3-7 天内 |
| 上下文 | 1 条/会话 | 每次对话 |
| 关注 | 2 条/次 | 冷却期 |
| 总结 | 1 次/周 | 7 天 |

### 去重规则

```typescript
// 内容哈希去重
const contentHash = hash(type + content);

// 时间窗口去重
if (lastPushed.has(contentHash) && withinTimeWindow(contentHash)) {
  return; // 跳过
}

// 会话级别去重
if (currentSession.pushed.has(contentHash)) {
  return; // 跳过
}
```
