# Corivo

## 有一个人，一直在替你操心

你有没有过这种感觉——

开完会，总觉得漏了什么。跟人聊完，隐约记得对方说了个重要的事，但想不起来了。做方案的时候，感觉之前踩过类似的坑，但找不到当时的结论。

不是你记性差。是你要操心的事情太多了。

每天几十轮 AI 对话、几百条消息、几十个决策。散落在 Claude Code 里、飞书里、Cursor 里。你的大脑不可能全记住，这些工具也不会帮你记——对它们来说，每次对话都是第一次见面。

**如果有一个人，默默帮你记着这些事呢？**

不需要你说"帮我记一下"。不需要你整理笔记。不需要你在十个工具之间翻找。

它就在你的工作流里，安安静静地待着。你在 Claude Code 里做了一个技术选型，它记住了。你在飞书里跟同事约了个截止日期，它记住了。你在 Cursor 里踩了一个坑，它记住了。

然后在你需要的时候，它出现了：

> `[corivo] 张三说的反馈截止日期是明天，他还没回复。`

> `[corivo] 你上次在 project-B 里遇到过同样的问题，当时改用了连接池。`

> `[corivo] 你妈妈的生日是后天。去年你送的是鲜花。`

你自己都忘了的事，它替你记着。你还没意识到的关联，它已经帮你想到了。

---

## 它不是一个软件

你不需要打开它。它没有界面。没有仪表盘。没有待办列表。

它融入你已经在用的工具里——Claude Code、Cursor、飞书、Slack。它不是又一个需要你去维护的系统，而是你现有系统的一部分，让它们突然变得更懂你了。

安装只需要一行命令。之后你几乎不会意识到它的存在——直到它在某个关键时刻出现，带着一条你恰好需要的信息。

那个时候你会想：还好有它在。

---

## 它怎么越来越懂你

第一天，它从你的项目文件里了解你的技术偏好和工作风格。

第一周，它开始记住你在 AI 对话中做的每一个决策。

第一个月，它知道你跟每个合作者之间的约定，知道你哪些项目在推进、哪些搁置了，知道你做事的模式。

三个月后，它比你自己更清楚你的工作全貌。

不是因为它更聪明。是因为**你操心的事太多，会忘。它不会。**

而且它不只是被动地记着。它会替你想：

这两条信息是不是矛盾了？那个承诺是不是快到期了？你最近三个项目都选了同一个技术栈，要不要固定下来？

它不是在执行指令。它是在**关心你的事**。

---

## 但有一件事，它绝对不做

**它不看你的数据。**

你的一切信息只存在你自己的设备上。多设备同步全程端到端加密——同步服务器只经手密文，连 Corivo 团队都无法解密。

项目完全开源。每一行代码可审计，你也可以自建服务器。

它替你操心，但不窥探你。

---

## 开始

```bash
npm install -g corivo
corivo init
```

不需要新软件、新界面、新习惯。

只需要让它进入你的工作流。然后忘掉它。

它会在你需要的时候出现。
