# Corivo · 设计文档 v1.0

> Agent-first 个人记忆层。采集无感，更新自动，推送有感。

---

## Corivo 是什么

Corivo 是一个嵌入用户已有工作流的**个人记忆层**。它不是一个独立的 App，而是寄生在 Claude Code、Cursor、飞书等工具中的后台服务——自动从用户的 AI 对话和消息中采集信息，持续整理和更新，在合适的时机以 `[corivo]` 的名义主动提醒用户。

用户不需要下载新软件、学习新界面、改变任何习惯。安装一个 CLI，跑一次 `corivo init`，记忆层就开始工作了。

---

## 解决什么问题

我们每天跟 AI 对话、在飞书里沟通、在代码里做决策。这些信息散落在几十个工具里，用完即忘。下一次需要时，要么想不起来，要么花时间翻历史记录。

Corivo 把这些散落的信息自动汇聚成一个统一的记忆库，让 AI 工具在帮你干活时能调用你的全部历史上下文——你做过的决策、你的偏好、你跟每个人说过的关键内容。

---

## 核心理念

**文本是唯一的"源码"。** Agent 能处理，人能直接读。所有结构化索引都是文本的派生物，不是本体。

**人是读者，Agent 是管家。** 记录、整理、查询、更新——全部由 Agent 执行。人只在需要时阅读结果。

**自然语言即编程语言。** 用户说一句"以后周报帮我这样写"，Agent 将其存为指令，下次自动执行。零行为改变。

**安全是基础。** 所有数据只存在用户本地设备上，端到端加密同步。Corivo 团队无法读取用户数据。项目开源，可审计、可自托管。

---

## 核心流程

```
采集（无感）──→ 更新（自动）──→ 推送（有感）
                    ↑                  │
              记忆生命周期              │
              （衰减/整合/重构）        │
                                      ↓
        ← 用户在已有工具中自然交互 ←──┘
```

**采集**：通过规则注入让 AI 工具主动存储信息，通过事件订阅监听消息平台。用户无感。

**更新**：心跳引擎持续处理新数据、整合重复内容、衰减过时信息。用户无感。

**推送**：在对话中以 `[corivo]` 的名义主动提醒相关记忆。用户有感，且知道是 Corivo 在帮忙。

---

## 产品定位

Corivo 是守夜人（主动 Agent）和 Mesh（跨工具上下文注入）的共同底层——一个 Agent 和人共享的记忆层。

- 守夜人负责感知和行动，Corivo 负责记忆
- Mesh 负责跨工具串联，Corivo 提供统一的知识存储
- 先从个人工作流增强场景切入
- 开源项目，用户数据完全自主可控

## 技术选型概览

| 层面 | 选型 | 理由 |
|------|------|------|
| 本地存储 | SQLCipher（加密 SQLite） | 全平台、毫秒级、离线可用 |
| 语义搜索 | sqlite-vss / LLM 直接匹配 | 个人数据量级足够 |
| 多设备同步 | E2EE 中继服务 | 安全等价 P2P，体验等价云同步 |
| 对外接口 | CLI（核心）+ MCP Server + IM Bridge | 覆盖 CLI Agent、GUI AI 工具、消息平台 |
| 主动接入 | 规则注入（CLAUDE.md 等） | 零进程、零协议，AI 工具自己使用 Corivo |
| 心跳运行 | 寄生在已有常驻进程 | 不引入新进程，用户无感 |
